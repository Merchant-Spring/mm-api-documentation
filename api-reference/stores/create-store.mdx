# Create store

Creates a new store associated with the given organisation.

<Info>
  This endpoint currently returns dummy data. Real data access will be wired in a future release.
</Info>

## Endpoint

```text
POST /organisations/{organisationId}/stores
```

## Authentication

All requests must include your API key in the `x-api-key` header. See [Authentication](/authentication) for details.

## Path parameters

<ParamField path="organisationId" type="string" required>
  The unique identifier of the organisation to create the store under.
</ParamField>

## Request body

The request body must be a JSON object with the following fields.

<ParamField body="name" type="string" required>
  Display name for the store.
</ParamField>

<ParamField body="marketplace" type="string" required>
  The marketplace platform the store is connected to (e.g. `amazon`, `ebay`).
</ParamField>

## Request example

```bash
curl -X POST "https://api.marketplacemanager.com/v1/organisations/org-123/stores" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "My Amazon Store", "marketplace": "amazon"}'
```

## Response

Returns `201 Created` with an object containing the newly created store.

```json
{
  "store": {
    "storeId": "store-1749520000000",
    "organisationId": "org-123",
    "name": "My Amazon Store",
    "marketplace": "amazon"
  }
}
```

## Response fields

<ResponseField name="store" type="object">
  The created store object.

  <Expandable title="Store object fields">
    <ResponseField name="storeId" type="string">
      Unique identifier generated for the store.
    </ResponseField>
    <ResponseField name="organisationId" type="string">
      The organisation identifier from the request path.
    </ResponseField>
    <ResponseField name="name" type="string">
      Display name of the store.
    </ResponseField>
    <ResponseField name="marketplace" type="string">
      The marketplace platform the store is connected to (e.g. `amazon`, `ebay`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Meaning |
|--------|---------|
| `400`  | Missing or empty `organisationId` path parameter, missing or empty `name` or `marketplace` body field, or invalid JSON body |
| `401`  | Missing or invalid API key |
