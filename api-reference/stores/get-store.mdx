# Get store

Returns the details of a specific store belonging to an organisation.

<Info>
  This endpoint currently returns dummy data. Real data access will be wired in a future release.
</Info>

## Endpoint

```
GET /organisations/{organisationId}/stores/{storeId}
```

## Authentication

All requests must include your API key in the `x-api-key` header. See [Authentication](/authentication) for details.

## Path parameters

<ParamField path="organisationId" type="string" required>
  The unique identifier of the organisation the store belongs to.
</ParamField>

<ParamField path="storeId" type="string" required>
  The unique identifier of the store to retrieve.
</ParamField>

## Request example

```bash
curl -X GET "https://api.marketplacemanager.com/v1/organisations/org-123/stores/store-1" \
  -H "x-api-key: YOUR_API_KEY"
```

## Response

Returns a `store` object.

```json
{
  "store": {
    "storeId": "store-1",
    "organisationId": "org-123",
    "name": "Dummy Store",
    "marketplace": "amazon"
  }
}
```

## Response fields

<ResponseField name="store" type="object">
  The requested store.

  <Expandable title="Store object fields">
    <ResponseField name="storeId" type="string">
      Unique identifier for the store.
    </ResponseField>
    <ResponseField name="organisationId" type="string">
      The organisation the store belongs to.
    </ResponseField>
    <ResponseField name="name" type="string">
      Display name of the store.
    </ResponseField>
    <ResponseField name="marketplace" type="string">
      The marketplace platform the store is connected to (e.g. `amazon`, `ebay`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Meaning |
|--------|---------|
| `400`  | Missing or empty `organisationId` or `storeId` path parameter |
| `401`  | Missing or invalid API key |
| `404`  | Store not found (will apply once real data access is wired) |
