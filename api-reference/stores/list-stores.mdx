# List stores

Returns a list of all stores associated with the given organisation.

<Info>
  This endpoint currently returns dummy data. Real data access will be wired in a future release.
</Info>

## Endpoint

```
GET /organisations/{organisationId}/stores
```

## Authentication

All requests must include your API key in the `x-api-key` header. See [Authentication](/authentication) for details.

## Path parameters

<ParamField path="organisationId" type="string" required>
  The unique identifier of the organisation whose stores you want to retrieve.
</ParamField>

## Request example

```bash
curl -X GET "https://api.marketplacemanager.com/v1/organisations/org-123/stores" \
  -H "x-api-key: YOUR_API_KEY"
```

## Response

Returns an object containing the `organisationId` and a `stores` array.

```json
{
  "organisationId": "org-123",
  "stores": [
    {
      "storeId": "store-1",
      "name": "Dummy Store One",
      "marketplace": "amazon"
    },
    {
      "storeId": "store-2",
      "name": "Dummy Store Two",
      "marketplace": "ebay"
    }
  ]
}
```

## Response fields

<ResponseField name="organisationId" type="string">
  The organisation identifier from the request path.
</ResponseField>

<ResponseField name="stores" type="array">
  List of store objects belonging to the organisation.

  <Expandable title="Store object fields">
    <ResponseField name="storeId" type="string">
      Unique identifier for the store.
    </ResponseField>
    <ResponseField name="name" type="string">
      Display name of the store.
    </ResponseField>
    <ResponseField name="marketplace" type="string">
      The marketplace platform the store is connected to (e.g. `amazon`, `ebay`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Meaning |
|--------|---------|
| `400`  | Missing or empty `organisationId` path parameter |
| `401`  | Missing or invalid API key |
