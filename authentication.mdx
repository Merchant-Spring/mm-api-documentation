# Authentication

The Marketplace Manager API uses **API keys** for authentication. Every request must include your API key in the `x-api-key` request header.

## Getting your API key

1. Log in to your [Marketplace Manager dashboard](https://app.marketplacemanager.com)
2. Navigate to **Settings → API**
3. Click **Generate API Key**
4. Copy and store your key securely — it won't be shown again

<Warning>
  Keep your API key secret. Never expose it in client-side code or public repositories.
</Warning>

## Making authenticated requests

Pass your API key in the `x-api-key` header:

```bash
curl -X GET "https://api.marketplacemanager.com/v1/organisations/org-123/stores" \
  -H "x-api-key: YOUR_API_KEY"
```

## Rate limits

| Plan       | Requests per month |
|------------|--------------------|
| Starter    | 100,000            |
| Growth     | 500,000            |
| Enterprise | Unlimited          |

The API enforces per-second throttling as well:

| Limit       | Value |
|-------------|-------|
| Burst limit | 50 requests  |
| Rate limit  | 25 requests/second |

When you exceed the rate limit, the API returns a `429 Too Many Requests` response. Retry after the number of seconds indicated in the `Retry-After` header.

## Error codes

| Status | Meaning                                          |
|--------|--------------------------------------------------|
| `400`  | Bad request — missing or invalid parameters      |
| `401`  | Missing or invalid API key                       |
| `403`  | Valid key but insufficient permissions           |
| `429`  | Rate limit exceeded                              |
| `500`  | Internal server error — contact support          |
