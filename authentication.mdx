---
title: "Authentication"
description: "How to authenticate with the MerchantSpring Public API using your API key."
---

The MerchantSpring Public API uses **API keys** for authentication. Every request must include your key in the `x-api-key` request header.

## Getting your API key

API keys are **provisioned by MerchantSpring** — they are not generated from this documentation site.

1. Contact your MerchantSpring representative (or account manager) to request API access for your organisation.
2. You'll be issued an API key tied to your organisation and plan.
3. Store the key securely — treat it like a password.

<Warning>
  Keep your API key secret. Never embed it in client-side code, mobile apps, or public repositories. If a key is exposed, contact MerchantSpring to have it rotated.
</Warning>

## Making authenticated requests

Pass your key in the `x-api-key` header on every request:

```bash
curl -X POST "https://mm-api.merchantspring.io/channels" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{}'
```

<Note>
  You can also try any endpoint directly from the browser using the **playground** on each API Reference page — paste your key into the `x-api-key` field and press **Send**.
</Note>

## What your key determines

Your key is associated with:

- **Your organisation** — requests return data only for the channels your organisation can access.
- **Your plan / feature flags** — some endpoints require specific access enabled on your key. For example, Amazon orders and the Products report require the **Enterprise** plan; calling them without it returns `401`.

## Error responses

| Status | Meaning |
|--------|---------|
| `400`  | Bad request — missing or invalid parameters in the body or query. |
| `401`  | Valid key, but it lacks permission for this endpoint (e.g. an Enterprise-only endpoint). |
| `403`  | Missing or invalid API key. |
| `404`  | The requested resource (e.g. a `reportId`) was not found. |

<Info>
  Usage limits (request rate and volume) are governed by the usage plan attached to your API key. If you need higher limits, talk to your MerchantSpring representative.
</Info>
